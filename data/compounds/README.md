# Compound library

## Files

- **`seeds.json`** — hand-curated anchor compounds (pharma + herbs + supplements + biologics). The source of truth for the master library.
- **`MCAS_Compound_Library_v1.csv`** — generated by `scripts/build_compound_library.py`. SMILES auto-fetched from PubChem PUG-REST by CID.
- **`.smiles_cache.json`** — local PubChem response cache (gitignored).

## How to add a compound

1. Edit `seeds.json` — add an object to the `compounds` array.
2. Required keys: `name`, `category` (`rescue` | `maintenance` | `remission`), `mechanism`, `target`, `evidence_level`, `evidence_notes`, `source_refs`.
3. If it's a small molecule: include `pubchem_cid` (integer). SMILES will be auto-fetched.
4. If it's a peptide / antibody / enzyme / plant extract: set `pubchem_cid: null` and `biologic_flag` to one of `peptide_*`, `monoclonal_antibody_*`, `enzyme`, `enzyme_mixture`, `plant_extract`. These bypass SMILES fetch and are flagged in downstream pipelines.
5. Rerun the build: `python scripts/build_compound_library.py`.
6. Validate: `python scripts/validate_smiles.py`.

## Schema

`MCAS_Compound_Library_v1.csv` columns:

| Column | Description |
|--------|-------------|
| `name` | Common name |
| `pubchem_cid` | PubChem Compound ID (empty for biologics) |
| `smiles` | Isomeric SMILES from PubChem (empty for biologics) |
| `canonical_smiles` | RDKit-canonicalized SMILES |
| `category` | `rescue` / `maintenance` / `remission` |
| `subcategory` | Drug class / natural-product family |
| `mechanism` | Short mechanism summary |
| `target` | Primary druggable target |
| `evidence_level` | `high` / `medium` / `low` |
| `evidence_notes` | Brief evidence summary |
| `source_refs` | Comma-separated citation tags |
| `biologic_flag` | If non-small-molecule, the kind of biologic |
