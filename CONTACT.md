## How to reach us

All contact is routed through GitHub so the audit trail is public and
contributors can see what's being asked, by whom, with what response.

| Why you're reaching out | Where to go |
|---|---|
| General question | [GitHub Discussions](https://github.com/mrdulasolutions/MCAS.Opensource/discussions) or a [general issue](https://github.com/mrdulasolutions/MCAS.Opensource/issues/new) |
| Add a compound to the library | [Compound suggestion issue](https://github.com/mrdulasolutions/MCAS.Opensource/issues/new?template=compound_suggestion.md) |
| Report a trigger pattern (anonymous) | [Trigger report issue](https://github.com/mrdulasolutions/MCAS.Opensource/issues/new?template=trigger_report.md) |
| Propose a hypothesis | [Hypothesis proposal issue](https://github.com/mrdulasolutions/MCAS.Opensource/issues/new?template=hypothesis_proposal.md) |
| Academic / lab collaboration | [Academic collaboration issue](https://github.com/mrdulasolutions/MCAS.Opensource/issues/new?template=academic_collaboration.md) |
| Nonprofit / foundation partnership | [Nonprofit partnership issue](https://github.com/mrdulasolutions/MCAS.Opensource/issues/new?template=nonprofit_partnership.md) |
| Wet-lab pre-registration | [Wet-lab pre-registration issue](https://github.com/mrdulasolutions/MCAS.Opensource/issues/new?template=wet_lab_preregistration.md) |
| Press / media | Open an [issue tagged `press`](https://github.com/mrdulasolutions/MCAS.Opensource/issues/new?labels=press) — we respond within 48 hours |
| Patient-safety report (urgent) | Open an [issue tagged `safety`](https://github.com/mrdulasolutions/MCAS.Opensource/issues/new?labels=safety) — acknowledgement within 24 hours, see [SECURITY.md](SECURITY.md) |
| Software vulnerability | Private disclosure via [Security Advisory](https://github.com/mrdulasolutions/MCAS.Opensource/security/advisories) — see [SECURITY.md](SECURITY.md) |
| Code of Conduct concern | [Issue tagged `code-of-conduct`](https://github.com/mrdulasolutions/MCAS.Opensource/issues/new?labels=code-of-conduct) — see [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) |

For sensitive matters that need a private channel, open an issue and
request a private email handoff; we will reply from a verified address
linked from the GitHub Organization profile.

## What we will never ask for

- Personally identifying patient information.
- Payment, credentials, keys, or tokens.
- Exclusivity over hypotheses, compounds, or data published here.
- IP transfer.

## Authority

Decisions on merging contributions, accepting partnerships, and
enforcing the Code of Conduct rest with MR Dula Medical until / unless
this project transitions to a fiscal sponsor or successor governance
structure (see [ROADMAP.md](ROADMAP.md) and
[audiences/for-nonprofits.md](audiences/for-nonprofits.md)).

## Mailing address (for legal correspondence only)

For legal correspondence that cannot be handled via GitHub, request a
mailing address by opening a private channel as described above. We
deliberately do not publish a postal address inline because (a) the
public-issue routes are sufficient for nearly all interactions, and
(b) it discourages unsolicited mail that doesn't belong in a research
project. Legal counsel inquiries always get a response.
